leroy.dev Portfolio

Fresh out of Dev Academy, I've taken on the exciting task of building a portfolio website. I'm applying my newfound skills and confidence to complete this mission-critical project.

Tech |

Stage One: Learning Outcomes

Figma, HTML and CSS

As I embarked on developing my portfolio, I also set some learning objectives. I aimed to refine my knowledge about Figma, HTML, CSS, and gain an overview of some popular frameworks like SASS and Tailwind. With these goals in mind, I organized complementary online courses to augment my learning.

Why, how and expectations

Transitioning from a professional design background to software development, I understand the immense value of presentation. A strong, fundamental grasp of the tools essential for visual presentation is important to me. Design tools like Figma offer an alternative way to communicate early-stage ideas to my team visually.

In terms of the building blocks of the web, HTML and CSS, I wanted to improve my skills related to visual structure, responsive design, animations, and features. Furthermore, I aimed to learn best practices concerning accessibility, as well as folder and file organisation.

To bridge any knowledge gaps, I supplemented my foundational skills with SkillShare online classes, such as "Figma UI/UX Design Essentials" by Daniel Scott, an Adobe Certified Trainer.

Goals and takeaways

By the end of this journey, my goal is to have a strong understanding of front-end development, from start to finish. I aim to be confident enough to discuss and offer my opinions on the design, code, and structure of websites.

Stage Two: Planning

Design and Structure

Project Details My portfolio website is built with HTML, CSS, and a dash of JavaScript for interactivity. I plan to include a home page, a portfolio section with case studies, a blog. The portfolio section will showcase my design work and any coding projects I've completed.

With a background in design, I have a keen eye for aesthetics, balance, and user experience, which I believe gives me a unique perspective in web development. I'm not just coding; I'm crafting experiences.

For this project, I'm trying to stick to best practices as much as possible using semantic HTML tags, commenting my code for clarity, and ensuring that the site is accessible.

Stage Three: Conclusion and Future Plans

Reflection

This journey is already teaching me so much, not just about coding but about how to approach problems and how to think critically.

Next Steps

My next big leap will be diving into JavaScript frameworks, starting with React, to add more dynamic features to my portfolio and future projects.
